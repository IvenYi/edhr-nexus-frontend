# eDHR 系统双栏侧边栏 UI 重设计规范

> 版本：1.0
> 日期：2026-06-05
> 适用范围：eDHR 电子批次记录系统 — 双栏侧边栏（模块栏 + 功能菜单栏）及 Header

---

## 一、当前设计问题诊断

### 1.1 配色体系缺乏行业辨识度

| 问题 | 分析 |
|------|------|
| 模块栏 `#1a1a2e` 是一种深暗蓝紫色，偏向科技/互联网风格 | 制药/医疗行业应传递「洁净、精密、可信赖」的气质，而非暗黑科技感。该色饱和度低且偏紫，既不够沉稳也不够专业 |
| 激活色 `#1890ff` 是 Ant Design 默认蓝色 | 直接使用框架默认色，毫无品牌辨识度。在制药行业中，纯蓝过于冰冷，缺少「生命关怀」的温度感 |
| 功能菜单栏 `#fafafa` 与内容区 `#f5f5f5` 对比不足 | 两区域色差仅 5 个灰度值，视觉边界模糊，用户无法快速区分导航区与内容区 |

### 1.2 对比度与可访问性缺陷

| 问题 | 分析 |
|------|------|
| 模块栏未选中态 `#8c8c8c` on `#1a1a2e` 对比度仅约 3.2:1 | WCAG AA 标准要求 4.5:1，当前未达标，低视力用户难以辨认 |
| 功能菜单栏文字 `#595959` on `#fafafa` 对比度约 5.7:1 | 勉强达标但偏低，且灰文字缺乏层级区分 |
| 选中态 `rgba(24,144,255,0.15)` on `#1a1a2e` 背景极其微弱 | 15% 透明度的蓝色在深色背景上几乎不可见，选中与未选中的视觉差异不够明确 |

### 1.3 层次感与空间比例失衡

| 问题 | 分析 |
|------|------|
| 模块栏 64px 宽度过窄 | 4 个图标+标签挤在 64px 内，图标 22px + 文字 11px 的组合过于局促，标签如「生产」「系统」容易截断或换行不美观 |
| 功能菜单栏 200px 偏窄 | 子菜单项如「工厂/车间/产线」在 200px 内 `pl:4`（32px 缩进）后可用空间不足 160px，长文字被迫截断 |
| 模块栏图标与标签垂直堆叠，间距仅 `mb: 0.5`（4px） | 图标与标签之间视觉粘连，不够舒展 |
| 菜单项 `minHeight` 未显式指定，默认 48px | 对制药行业的操作环境（可能戴手套操作），48px 触控目标偏小 |

### 1.4 选中态指示器设计粗糙

| 问题 | 分析 |
|------|------|
| 右侧白色竖条 3px × 高度50% | 指示器与模块栏背景色缺乏关联，白色竖条在 `#1a1a2e` 上显得生硬、割裂，像临时方案 |
| 功能菜单栏子菜单选中仅靠背景色 `rgba(24,144,255,0.08)` 区分 | 8% 透明度几乎不可察觉，用户很难确认当前所在位置 |

### 1.5 品牌感缺失

| 问题 | 分析 |
|------|------|
| 整体视觉与「通用后台模板」无异 | 没有任何制药/医疗行业的视觉线索——颜色、形状、细节均无行业特征 |
| Header 使用 MUI 默认 `#1976d2` 蓝色 AppBar | 与模块栏深色、功能菜单栏浅色形成三段割裂的色块，缺乏整体色彩统筹 |
| 系统名「eDHR 系统」直接使用 `h6` Typography | 缺少品牌标识（Logo / icon），纯文字标题没有视觉记忆点 |

### 1.6 交互细节不足

| 问题 | 分析 |
|------|------|
| 模块切换无过渡动画 | 点击模块后功能菜单栏内容瞬间切换，视觉跳变 |
| 菜单折叠/展开使用 MUI 默认 Collapse | 无自定义缓动，展开后子菜单项无 stagger 入场效果 |
| 悬停态 `rgba(255,255,255,0.05)` 几乎不可见 | 深色背景上 5% 白色叠加无法提供有效反馈 |

## 二、重设计方案

### 2.1 设计原则

1. **行业气质优先**：配色、形态传递制药行业的「洁净、精密、可信赖」，而非互联网产品的「活泼、酷炫」
2. **清晰的信息层级**：模块栏 > 功能菜单 > 子菜单，三级导航通过背景明度、文字色阶、指示器粗细形成明确层次
3. **高可访问性**：所有文字/背景组合至少达到 WCAG AA（4.5:1），关键交互元素达到 AAA（7:1）
4. **触控友好**：制药车间环境可能戴手套操作，所有可点击区域最小 44px
5. **克制的品牌表达**：通过色彩选择和微妙的形态细节传达行业感，不堆砌医疗图标或装饰

### 2.2 配色体系

#### 核心色板

```
品牌主色（Primary）       #0D6EAA  — 医疗蓝，取自制药实验室标准色，沉稳且具辨识度
品牌辅色（Secondary）     #00897B  — 青绿色，暗示生命与洁净，用于辅助强调
中性深色（Neutral Dark）  #1B2A4A  — 海军蓝黑，模块栏背景，比纯黑有温度
中性浅色（Neutral Light） #F7F9FC  — 冷白偏蓝，功能菜单栏背景，与内容区有区分
内容区背景               #EFF2F7  — 带蓝调的浅灰，与菜单栏拉开层次
警告/强调色              #E65100  — 深琥珀橙，用于通知、关键操作，避免红蓝冲突
```

### 2.4 模块栏详细设计

#### 模块项布局

```
┌──────────────────────────┐
│                          │
│       ┌──────────┐       │
│       │  Icon    │       │  22×22px
│       │  22×22   │       │
│       └──────────┘       │
│         ↕ 6px            │
│        Label             │  11px
│                          │
└──────────────────────────┘
     72px 宽 × 64px 高
```

#### 选中态指示器

**放弃右侧白色竖条方案**，改用**左侧品牌色指示条**：

- 位置：模块项左侧边缘
- 尺寸：宽 3px × 高 32px（居中于模块项，上下各留 16px）
- 颜色：`#0D6EAA`（品牌主色）
- 圆角：右侧 2px
- 选中态背景：`rgba(13, 110, 170, 0.20)` 覆盖整个模块项

**理由**：左侧指示条与功能菜单栏的左侧指示条形成统一的视觉语言；品牌色指示条比白色竖条更有品牌辨识度，同时避免了深色背景上白色竖条的生硬感。

#### 悬停动效

```
默认态  →  悬停态
背景: transparent  →  rgba(255,255,255,0.08)
过渡: 150ms ease-out

选中态  →  悬停态
背景: rgba(13,110,170,0.20)  →  rgba(13,110,170,0.28)
过渡: 150ms ease-out
```

图标在悬停时微放大：`transform: scale(1.08)`，过渡 150ms ease-out。

### 2.5 功能菜单栏详细设计

#### 菜单标题项

```
┌────────────────────────────────────────┐
│  Icon  20px   Label         ▸/▾       │
│  ←12px→      14px/600    18px arrow   │
│  ←──20px──→                  ←16px─→  │
└────────────────────────────────────────┘
         220px 宽 × 44px 高
```

- 默认态：文字 `#2C3E56`，背景透明
- 悬停态：背景 `rgba(13, 110, 170, 0.05)`，过渡 150ms ease-out
- 选中态：文字 `#0D6EAA`，字重 600
- 展开箭头：`#5A6F88`，展开时旋转 180 度

#### 子菜单项

```
┌────────────────────────────────────────┐
│ ┃  SubMenu Label                       │
│ ┃  3px                                  │
│ ←4px→←──40px──→         13px/400      │
└────────────────────────────────────────┘
         220px 宽 × 40px 高
```

- 左侧指示条：
  - 选中态：宽 3px × 高 24px，颜色 `#0D6EAA`，距左侧 8px，垂直居中，圆角 2px
  - 未选中态：不显示
- 默认态：文字 `#4A5E76`，背景透明
- 悬停态：背景 `rgba(13, 110, 170, 0.05)`，过渡 150ms ease-out
- 选中态：文字 `#0D6EAA`，字重 500，背景 `rgba(13, 110, 170, 0.08)`

#### 子菜单层级视觉区分

当前系统为两级菜单（标题 → 子菜单），如未来扩展至三级：

| 层级 | 左缩进 | 字号 | 字重 | 文字色 |
|------|--------|------|------|--------|
| 菜单标题 | 20px | 14px | 400/600 | `#2C3E56` / `#0D6EAA` |
| 二级子菜单 | 44px | 13px | 400/500 | `#4A5E76` / `#0D6EAA` |
| 三级子菜单（预留） | 64px | 12px | 400/500 | `#6B7D92` / `#0D6EAA` |

#### 折叠/展开交互

- 箭头图标旋转动画：`transform: rotate(0deg)` → `rotate(180deg)`，200ms ease-in-out
- 子菜单列表展开：使用 MUI `Collapse` 组件，`timeout={200}`，`easing="cubic-bezier(0.4, 0, 0.2, 1)"`
- 子菜单项 stagger 入场：每项延迟 30ms，`opacity: 0 → 1`，`transform: translateX(-4px) → 0`，150ms ease-out

### 2.6 Header 详细设计

#### 整体结构

```
┌──────────────────────────────────────────────────────────────┐
│  [☰]  [💊 Logo]  eDHR 系统            🔔  张三 ▾           │
│  ←20px─→ ←10px→                       ←8px→ ←16px→         │
└──────────────────────────────────────────────────────────────┘
              56px 高 · 白色背景 · 底部 1px 边框
```

**关键变更**：Header 从蓝色 AppBar 改为白色背景 + 底部边框。

**理由**：
- 白色 Header 与模块栏深色背景形成明确的视觉分层——Header 属于「全局工具栏」，不属于导航
- 消除蓝色 AppBar 带来的三色割裂感（蓝 Header + 深色模块栏 + 浅色菜单栏）
- 白色 Header 使系统标题、通知、头像等元素更易阅读

#### 系统标题区

- Logo 图标：使用制药行业关联图标（胶囊/盾牌/十字结合），尺寸 28×28px，颜色 `#0D6EAA`
- 系统名称：`eDHR`（品牌主色 `#0D6EAA`，字重 700）+ 空格 + `系统`（`#2C3E56`，字重 400）
- 字号：16px
- 效果示意：**eDHR** 系统

#### 收起/展开按钮

- 图标：`MenuRounded`（展开时）/ `ChevronLeftRounded`（收起时）
- 尺寸：36×36px 按钮区域，图标 20px
- 颜色：默认 `#5A6F88`，悬停 `#0D6EAA`
- 悬停背景：`rgba(13, 110, 170, 0.08)`，圆角 8px
- 过渡：150ms ease-out

#### 通知区域

- 铃铛图标：24px，颜色 `#5A6F88`
- 未读计数徽章：背景 `#E65100`（深琥珀橙），文字白色，尺寸 16×16px，最小宽度 16px
- 悬停：图标颜色变为 `#0D6EAA`

#### 用户头像

- 尺寸：32×32px 圆形
- 背景色：`#00897B`（品牌辅色青绿）
- 文字：白色，14px，取用户名首字
- 悬停：外圈 2px `rgba(13, 110, 170, 0.3)` 描边，过渡 150ms

#### 用户下拉菜单

- 背景：白色
- 阴影：`0 4px 16px rgba(27, 42, 74, 0.12)`
- 圆角：8px
- 菜单项高度：40px
- 菜单项字号：14px
- 用户名显示行：字重 500，`#2C3E56`
- 分隔线：1px `#D6DEE8`
- 退出登录项：悬停背景 `rgba(230, 81, 0, 0.06)`，文字 `#E65100`

### 2.7 动效与过渡

#### 侧边栏收起/展开

```css
/* 功能菜单栏宽度过渡 */
transition: width 280ms cubic-bezier(0.4, 0, 0.2, 1);

/* 内容区左边距过渡 */
transition: margin-left 280ms cubic-bezier(0.4, 0, 0.2, 1);
```

收起时功能菜单栏内容保持固定宽度 220px（内层容器不缩放），仅外层 `width` 从 220px 过渡到 0px，`overflow: hidden` 裁切内容，避免文字重排。

#### 模块栏切换

切换模块时，功能菜单栏内容使用淡入效果：

```css
/* 旧内容淡出 */
opacity: 1 → 0, 120ms ease-in

/* 新内容淡入 */
opacity: 0 → 1, 180ms ease-out
```

#### 菜单折叠/展开

```css
/* MUI Collapse 配置 */
timeout={200}
easing="cubic-bezier(0.4, 0, 0.2, 1)"

/* 子菜单项 stagger 入场 */
opacity: 0 → 1
transform: translateX(-4px) → translateX(0)
duration: 150ms
delay: index × 30ms (max 5 items, then reset)
easing: ease-out
```

#### 悬停反馈

| 元素 | 属性 | 过渡时间 | 缓动 |
|------|------|----------|------|
| 模块项背景 | background-color | 150ms | ease-out |
| 模块项图标 | transform (scale) | 150ms | ease-out |
| 菜单项背景 | background-color | 150ms | ease-out |
| Header 按钮 | color, background-color | 150ms | ease-out |
| 展开箭头 | transform (rotate) | 200ms | ease-in-out |

### 2.8 品牌感细节

#### 色彩策略

- **医疗蓝 `#0D6EAA`** 作为品牌主色，取代通用蓝 `#1976d2`。该色取自制药实验室仪器面板的标准蓝色域，比纯蓝更深沉、偏青，具有「精密仪器」的暗示
- **青绿 `#00897B`** 作为辅色，用于头像背景、关键状态标签，暗示「生命、洁净、安全」
- **暖橙 `#E65100`** 仅用于通知/警告，与冷色主调形成互补，确保关键信息不遗漏

#### 形态语言

- 圆角统一使用 2px（指示条）、8px（按钮/卡片/下拉），避免大圆角的互联网感，保持工业级系统的严谨
- 阴影克制：仅在 Header 下拉菜单使用 `0 4px 16px rgba(27,42,74,0.12)`，其余元素不使用阴影
- 指示条设计（3px 宽、左右居中对齐）统一应用于模块栏和子菜单选中态，形成「精准标记」的视觉隐喻——如同仪器面板上的校准标记

#### 字体策略

- 西文：`'Inter', 'Roboto', sans-serif` — Inter 的等宽数字特性适合显示批次号、编码等工业数据
- 中文：`'Noto Sans SC', sans-serif`
- 数字（如批次号、编码）推荐使用等宽特性：`font-variant-numeric: tabular-nums`

#### 微妙的行业暗示

- 系统标题「eDHR」使用品牌主色加粗，暗示这是系统品牌而非普通标签
- 模块栏底部可放置一个小型版本号（如 `v2.1`），使用 `#4A5E76`、10px 字号，体现 GMP 系统的版本管控文化
- 功能菜单栏顶部可显示当前模块名称（如「数据管理中心」），使用 12px、`#7B8FA8`、letter-spacing 0.5px 的大写风格，类似仪器面板的功能区标签
