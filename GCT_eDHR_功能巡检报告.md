## 1. 总体结论

`GCT eDHR` 是以 GMP 场景为中心的电子批记录 / 电子 DHR / 生产检验放行一体化系统。系统结构分成三层：

1. 基础建模层：定义车间、产品、设备、模板、工序路线、原因、审批流程、标签和解析规则。
2. 业务执行层：承接工单、批次 / SN 生产、返工、生产事务、检验执行、放行、表单填报、DHR 汇总与审核。
3. 治理追溯层：提供报表、看板、自定义报表、权限、集成、审计追踪、系统配置和模型字段设置。

核心业务闭环是：

基础建模 -> 工单 / 批次 / SN 准备 -> 生产执行 / 返工 / 事务处理 -> 检验执行 / 检验事务 -> 放行 -> 表单与 DHR 归档 -> 打印 / 下载 -> 追溯报表 / 看板 / 审计。

## 4. 端到端作业流程

### 4.1 新产品 / 新工艺上线流程

1. 基础建模中维护车间、产品家族、产品列表、单位、设备类型、设备列表。
2. 上传或新建 SOP 文档、表单模板、DHR 模板、流转单模板。
3. 配置编码规则、工序、工艺路线、制程配置。
4. 配置不良 / 报废原因、审核流程、标签参数 / 标签模板 / 解析规则。
5. 配检验方法、检验项目、检验分类、检验方案、检验规程。
6. 配生产、检验、放行事务列表和事务配置。
7. 试建工单、批次 / SN，验证生产执行、检验执行、放行、DHR 汇总链路。

### 4.2 批次生产流程

1. 生产准备中创建或导入工单。
2. 工单拆分成批次，批次进入批次管理。
3. 生产人员扫码或输入生产批次进入批次生产执行。
4. 系统根据当前工序、工艺路线、SOP、事务配置加载执行界面。
5. 执行报工、消耗、不良、报废、异常事务。
6. 如需返工，进入返工执行和返工列表。
7. 批次完成后结束，并进入 DHR / 检验 / 放行。

### 4.3 SN 生产流程

1. 工单生成 SN。
2. 扫码或输入 SN 进入 SN 生产执行。
3. 执行当前工序、事务、返工和 SN 事务。
4. SN 完成后关联检验、放行、DHR。

### 4.4 检验放行流程

1. 检验建模完成方法、项目、分类、方案。
2. 检验配置中将检验规程绑定产品 / SOP / 检验表单 / 事务。
3. 检验执行按批次 / SN 和检验类型加载方案。
4. 检验列表记录流水号、类型、批次 / SN、工单、产品、数量、状态。
5. 检验事务处理完成后，放行列表可查看 DHR 并执行放行。

### 4.5 表单 / DHR / 记录本流程

1. 表单填报中新建或填报表单。
2. 表单审核处理待审记录，表单列表查看归档记录。
3. DHR填报按批次 / SN 查询；未查询到时可默认新建该批次 DHR。
4. DHR汇总将待汇总批次 / SN 汇总成 DHR 包。
5. DHR审核处理后进入 DHR列表。
6. 记录本用于周期性或部门级记录；记录本标签用于分类管理。
7. 变更管理处理表单、DHR、记录本变更。
8. 记录打印产生我的打印和打印列表，可批量下载或下载单个文件。

## 5. 做到什么程度算成功

### 5.1 MVP 成功标准

- 菜单完整：实现 8 个一级模块和 99 个叶子页面，路由、权限、面包屑 / 标签页、页面标题正确。
- 基础数据能闭环：产品、设备、模板、工序、工艺路线、原因、流程、标签均可新增、编辑、查询、状态管理和追溯。
- 执行业务能闭环：工单生成批次 / SN，生产执行能加载当前工序，事务能创建 / 处理 / 结束。
- 检验放行能闭环：检验方案能绑定产品和检验表单，检验执行产生记录，放行能引用检验和 DHR。
- 记录能闭环：表单填报、审核、列表；DHR填报、汇总、审核、列表；打印任务可追踪。
- 追溯能闭环：按工单、批次 / SN、产品、时间查询生产、检验、放行、表单、DHR、报工、报废、消耗、消息、标签记录。
- 审计合规：所有新增、编辑、删除、审批、放行、打印、下载、导出、登录、权限变更都写审计日志。

### 5.2 可上线成功标准

- 权限：角色、组织、用户席位、按钮权限、数据权限全部生效。
- 版本：SOP、表单模板、DHR模板、工艺路线、检验方案、解析规则等版本化对象可复制、创建版本、禁用旧版本、保留历史。
- 状态机：草稿、启用、禁用、执行中、待审核、已审核、已完成、已放行、已作废等状态转移受控。
- 数据完整性：关键业务记录不可物理删除；删除动作有引用校验；电子签名和审批记录不可篡改。
- 性能：列表分页、查询、导出、看板在大数据量下可用。
- 可验证：提供种子数据、端到端自动化测试、审计追踪校验、权限矩阵测试、异常流程测试。

```text
实现 GCT eDHR 的基础建模模块。要求支持车间、产品家族、产品、SOP、单位、设备、表单模板、DHR模板、流转单模板、编码规则、工序、工艺路线、制程配置、不良 / 报废原因、审核流程、标签参数、标签模板、解析规则。每类数据必须有列表、查询、分页、新建、编辑、详情、复制、版本创建、版本复制、删除 / 禁用、状态、建模追溯。模板类必须支持设计器入口；流程类必须支持流程配置；被生产、检验、放行、记录引用的数据不可破坏历史。
```
