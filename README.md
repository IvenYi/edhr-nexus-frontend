# edhr-nexus

医疗器械 eDHR 产品与工程仓库。

当前仓库首先用于沉淀产品需求、法规映射、领域模型和第一阶段合规基座设计。产品定位是面向国内医疗器械生产企业的 eDHR 系统，重点支持 2025 版《医疗器械生产质量管理规范》语境下的电子生产记录、检验记录、追溯、例外审核、产品放行、审计追踪、电子签名和归档导出。

## 第一阶段建议

优先开发合规可信基座和产品内核：

- 组织、用户、角色、权限
- 审计追踪
- 电子签名
- 受控记录存储
- 工作流/状态机
- 模板版本与记录实例关系
- DHR 实例聚合模型
- 最小完整性检查和放行阻断框架

后续生产、检验、追溯、放行等业务模块应复用上述基座能力，避免重复实现审计、签名和状态流转。
