# 组织管理 UI 标准

> 适用范围：组织管理菜单下的组织架构、岗位角色、用户管理，以及后续需要复用同类后台工作台形态的系统页面。

## 基准风格

- 使用 Vue Admin Plus 风格：`#1890ff` 主色、`#f6f8f9` 工作区、白色内容面板、`1px #e4e7ed` 边框、`5px` 圆角、紧凑 14px 字号。
- 页面优先服务内部产品/研发/系统管理员的高频操作，避免营销式卡片、重阴影、大圆角和装饰性背景。
- 内容区以“查询区 + 表格区 + 分页/统计 + 弹窗/抽屉”作为标准后台工作台结构。

## 查询区标准

- 查询项使用紧凑横向布局，常用字段优先：姓名、账号、状态、手机号、创建时间。
- 日期字段使用起止范围，鼠标点击不应选中文本；开始日期不得晚于结束日期，结束日期不得早于开始日期。
- 查询、重置按钮使用图标 + 文案，按钮高度与输入控件保持一致。
- 查询、重置按钮必须始终居右显示，不能因断点变化回到左侧。
- 查询区宽度必须受页面内容区约束，不得把 `main` 或 `body` 撑出横向滚动条。

## 表格标准

- 表格使用 `size="small"`、`stickyHeader`、固定布局和 `colgroup`。
- 表格外层容器必须承担横向滚动，页面级 `body`/`documentElement` 不应出现横向滚动条。
- 表头必须在表格容器内保持 sticky，数据行纵向滚动时表头始终显示在数据行上方。
- 表头高度统一为 48px，表头基础样式应挂在 `TableRow` 的 `.MuiTableCell-root` 上，避免选择框或默认 padding 把表头撑高。
- 标准用户列顺序：姓名、账号、手机号、所属组织、岗位角色、状态、创建人、创建时间。
- 字段列支持拖拽宽度，列宽按当前用户持久化，选择列和操作列固定宽度。
- 操作列固定在右侧，左侧分隔必须使用从表头到表体贯通的阴影效果，使用图标按钮和 Tooltip，不使用行内文字按钮。
- 阴影由表格容器或独立覆盖层统一绘制，覆盖操作列宽度，不得在每行操作单元格上重复绘制。
- 行点击打开右侧详情抽屉；复选框和操作按钮必须阻止事件冒泡。
- 组织字段显示完整路径，单行截断并通过 `title` 暴露完整文本。

## 分页标准

- 分页器固定使用“总数 + 分页 + 每页条数选择”布局，并放在表格面板底部。
- 每页条数选项统一为 20、50、100、200。
- 切换每页条数后必须回到第一页，避免页码越界导致表格空白。

## 用户表单标准

- 用户字段统一使用“账号”，不使用“用户名”。
- 新增/编辑弹窗使用 `DialogContent dividers`，字段控件高度为 40px，控件间距保持 12px 左右。
- 账号、姓名、所属组织、岗位角色为必填项。
- 编辑用户时账号禁用。
- 岗位角色为单选，但接口契约继续使用 `roleIds` 数组。
- 组织下拉显示公司/部门/班组路径，不显示组织编码。
- 邮箱和手机号保存前做格式校验；校验或后端错误用右上角 Snackbar 明确提示。

## 详情抽屉标准

- 行点击打开右侧全高抽屉，抽屉覆盖固定顶部栏，不预留固定 header 空白。
- 抽屉标题为“信息查看”，默认进入“数据信息” Tab。
- Tab 固定为“数据信息”和“数据审计”。
- 数据信息至少包含：基本信息、组织与角色、系统信息。
- 系统信息包含创建人、创建时间、更新人、更新时间。
- 首次创建时如果没有独立更新人或更新时间，更新人和更新时间显示创建人和创建时间。

## 审计标准

- 审计数据必须来自后端真实审计接口，不在前端伪造“当前快照”。
- 用户审计按 `entityType: USER_ACCOUNT` 和当前用户 ID 查询。
- 审计记录使用手风琴行，摘要展示操作人、操作动作、操作时间。
- 展开后按字段展示变更前/变更后内容；无审计记录时文案为“暂无审计记录”。

## 反馈和错误

- Snackbar 固定右上角。
- 保存、删除、移出、重置密码等动作必须显示成功或错误反馈。
- API 错误优先展示后端 message，不能只显示泛化的“保存失败”。

## 第一阶段复用策略

- 用户管理页先按组织架构页人员工作台的标准完成本页改造。
- 暂不抽大型通用表格组件，避免影响已经稳定的组织架构页。
- 后续当岗位角色、用户管理和组织架构三处都稳定后，再抽列宽、详情抽屉、审计手风琴、用户表单等小组件或 hook。
