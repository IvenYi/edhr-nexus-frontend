## 1. 产品概述

### 1.1 产品定位

本产品是一款**满足《医疗器械生产质量管理规范（2025版）》全部要求的合规软件基座**。它不是一个面向单一企业、单一场景的工具软件，而是一个可配置、可扩展、可租户化的**医疗器械质量管理数字化平台**。在此基座之上，初创团队可以快速定制和延伸面向不同类别医疗器械、不同规模企业的 SaaS 产品。

### 1.2 目标用户

| 用户类型 | 典型规模 | 核心场景 |
|---------|---------|---------|
| 医疗器械注册人（MAH） | 中小型创新企业 | 委托生产管理、上市放行、设计开发文档管理 |
| 医疗器械备案人 | 小型企业 | 备案产品全流程质量管理 |
| 受托生产企业 | 中大型制造企业 | 生产质量管理、批记录、生产放行 |
| 同时持有注册证并自行生产的企业 | 全规模 | 全覆盖的质量管理体系运行 |

### 1.3 核心价值主张

1. **法规全覆盖**：每一条 GMP 条款都有明确的功能映射，上线即合规。
2. **数智化基座**：从传统纸质/半电子化一步跃迁至 AI 驱动的质量管理平台。
3. **多租户原生**：一个平台支撑注册人、备案人、受托企业协同，跨租户委托生产关系天然打通。
4. **灵活部署**：云端 SaaS 快速开通，私有化离线部署满足数据主权和安全需求。
5. **可扩展生态**：基座提供标准 API 和插件机制，在此之上定制不同类别医疗器械的附录合规能力。

## 4. 多租户架构需求

### 4.1 租户隔离策略

| 隔离维度 | 策略 | 实现方式 |
|---------|------|---------|
| 数据隔离 | 物理隔离（独立Schema/数据库） | PostgreSQL Row-Level Security + Schema per Tenant |
| 配置隔离 | 逻辑隔离 | 每租户独立配置项集合，系统参数+业务参数分层 |
| 文件隔离 | 物理隔离 | 每租户独立Bucket/目录，访问密钥隔离 |
| 用户身份 | 逻辑隔离 | 租户ID绑定，JWT中包含租户上下文，所有API查询强制tenant_id过滤 |
| 审计日志 | 物理隔离 | 每租户独立日志表，租户间不可互相查阅 |
| 法规库 | 共享+租户定制 | 核心法规库共享只读，租户可添加自有法规/标准文件 |

## 6. 合规特色优势

### 6.1 相比传统QMS/DHR/MES的差异化

| 维度 | 传统方案 | 本软件基座 |
|------|---------|-----------|
| 法规覆盖度 | 分散在多个系统，覆盖率60-70% | 单一平台覆盖15章132条，100%法规映射 |
| 委托生产 | QMS缺乏跨企业协作机制 | 原生跨租户委托生产协同（双轨放行、联合变更、异常推送） |
| 风险管理 | 独立的FMEA工具，与实际业务脱节 | 嵌入式风险管理，在生产/变更/偏差等环节自动触发，不留死角 |
| UDI | 独立UDI系统 | 与批记录/追溯/召回深度集成 |
| DHF/DMR | 文档散落，追溯困难 | 设计开发全过程结构化管控，输入-输出-验证-确认追溯矩阵自动生成 |
| 计算机化系统验证 | 需额外服务 | 平台自带CSV验证包（离线部署），大幅降低验证成本 |
| 审计追踪 | 日志级，难以还原业务上下文 | 业务级审计追踪，操作者+操作内容+业务上下文+修改前后值完整记录 |
| 数智化 | 报表+看板 | AI驱动的趋势预警、异常检测、自动合规检查 |
| 多租户/SaaS | 单一企业部署 | SaaS原生多租户，创业团队零IT成本起步 |

## 7. 开放性问题

| 编号 | 问题 | 建议讨论方向 |
|------|------|------------|
| Q1 | 离线部署的CSV验证包是否需要第三方审计机构认证？ | 建议与药监局沟通确认，可能需要预留第三方审计接口 |
| Q2 | SaaS模式下，租户数据存储是否需要满足"境内存储"要求？ | 如果面向医疗器械企业，建议服务器全部在中国境内 |
| Q3 | 不同类别器械（无菌/植入/IVD/有源）的附录合规包是否在第一期？ | 第一期聚焦通用GMP规范（15章132条），附录包作为第二期插件化交付 |
| Q4 | 移动端（现场审核、扫码、离线采集）的开发优先级？ | 建议P1，与Web端核心功能同步开发，移动端是离线场景的关键入口 |
| Q5 | 平台本身是否需要通过ISO 13485认证？ | 建议作为公司级认证目标，但不影响第一版产品开发 |
| Q6 | 是否考虑对接药监局UDI数据库、不良事件监测系统等外部系统的API？ | 建议P1阶段开发对接，优先级：UDI数据库 > 不良事件监测系统 > 药监局监督检查 |
| Q7 | 多语言支持（中英文）？ | 国内GMP合规场景中文即可，如考虑出海再增加英文支持 |
